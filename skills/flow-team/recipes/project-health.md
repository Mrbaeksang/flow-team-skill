# 프로젝트 헬스체크

## 무엇
여러 프로젝트 중 어디가 새고 있는지 한눈에 — 관리자/리드용. 읽기 전용.

## 기대효과
- 여러 프로젝트의 **위험 신호를 한 표**로 — 어디부터 손볼지 즉시 판단
- 지연·담당없음 **비율**로 비교 → 규모 다른 프로젝트도 공정하게
- 위험 프로젝트는 **마감 트리아지/담당 배정 레시피로 바로 연결**

## 흐름 (API 조합)
1. `flow.myProjects()` → 전 프로젝트.
2. 각 `flow.tasks(pid, { pageSize: 100 })` → 집계: 미완료 / 지연(`END_DT<오늘` 미완료) / 담당없음 / 마감없음 수.
3. 프로젝트별 신호등(🔴지연多 / 🟡주의 / 🟢양호)으로 정렬해 위험순 제시.

## 응답 예시
```
📊 프로젝트 헬스 (위험순)
🔴 [Ent] 엔터AI/LLM 요건   미완료 8 · 지연 3 · 담당없음 2
🟡 사용자 피드백 루프       미완료 4 · 지연 1 · 담당없음 0
🟢 워크샵                   미완료 1 · 지연 0
→ "[Ent] 프로젝트 마감 트리아지 해줘" 로 연결
```

## 배운 것 (실호출 검증)
- **무거움:** 60개 프로젝트 × `tasks` = ~60콜 → 분당 120 한도. 429 자동 재시도가 처리하니 한 번에 돌려도 됨(단 직후 다른 대량 호출은 피함).
- **상태 카테고리:** `optionCategory`로 완료(2)/보류(3) 판정이 보드 무관하게 안전.
- **공유 프로젝트 주의:** 출퇴근기록 같은 회사 공통 프로젝트는 자동배정 업무로 카운트가 부풀 수 있음 — 신호등 해석 시 감안.
