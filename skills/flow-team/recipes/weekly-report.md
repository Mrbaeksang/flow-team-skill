# 주간 리포트

## 무엇
무엇이 끝났고, 진행 중이고, 지연됐는지 프로젝트별로 — 읽기 전용.

## 기대효과
- 한 주 성과를 **프로젝트별로 종합** — 주간 회의/보고 자료 즉석 생성
- 완료/진행/지연 **버킷**으로 한눈에
- (선택) 보고 프로젝트에 게시해 **팀 공유**

## 흐름 (API 조합)
1. `flow.myProjects()` → 활성 프로젝트 몇 개(또는 사용자에게).
2. 프로젝트별 `flow.tasks(pid, { pageSize: 100 })` → STATUS `optionCategory`로 분류(0대기·1진행·2완료·3보류). `END_DT`로 지연 판정.
3. 완료/진행/지연 버킷으로 프로젝트별 요약.

## 응답 예시
```
📊 주간 리포트 — 06-08 ~ 06-14
▸ [피플] 온보딩    완료 4 · 진행 2 · 지연 1
   - 지연: "상품기획실 안내" (마감 06-12)
▸ Flow 목표관리     완료 1 · 진행 1 · 지연 0
```

## 배운 것 (실호출 검증)
- **상태명은 보드마다 커스텀** — "운영 확인 완료" 같은 변형이 있어 이름만으로 버킷팅하면 샘. `optionCategory`(2=완료)로 묶는 게 안전.
- 필요하면 `flow.statusColumn(pid)`로 그 프로젝트의 옵션 전체(이름↔srno↔카테고리) 확인.
- 호출량은 헬스체크와 비슷(프로젝트당 1콜) — 429 재시도가 처리.

## 안전
- 읽기 전용. 게시하려면 `flow.createPost(...)` (확인 후).
