# 마감 일괄 부여

## 무엇
"마감 없는 업무"는 모든 마감 기준 판단에서 새어나간다. 한 번에 마감을 붙여 레이더에 올린다.

## 기대효과
- 브리핑·트리아지에서 **빠지던 업무들을 가시화** (마감 없으면 영영 안 보임)
- 일괄 제안으로 **한 번에 정리** — 하나씩 안 해도 됨
- 부여 후엔 **데일리 보고서에 자동 포함**

## 흐름 (API 조합)
1. `flow.myProjects()` → 프로젝트.
2. `flow.tasks(pid, { pageSize: 100 })` → 내 담당, 미완료, `END_DT` 없는 업무 목록.
3. 묶어서 제시하고 마감 제안(예: 1주 뒤, 또는 프로젝트별 일괄). 사용자가 조정.
4. 확인 후 각 `flow.setTaskEndDate(pid, taskId, "YYYYMMDD")`.

## 응답 예시
```
📅 마감일 없는 미완료 업무 37건. 일괄 제안:
- "사용자 재방문 유도 기능" 5건 → 06-23
- "온보딩 공지" 3건 → 06-20
당신: "그렇게 해줘" → 각 setTaskEndDate(...) ✓ (37건 설정)
```

## 배운 것 (실호출 검증)
- **마감 없는 업무가 의외로 많음:** 내 담당 미완료 중 37건이 마감 없음 → 그래서 이 레시피가 유용.
- `setTaskEndDate`는 `YYYYMMDD`(8자리), 검증됨. 업무는 항목별 1콜 → 많으면 시간 걸림(429 재시도가 처리).
- 되돌리려면 다시 `setTaskEndDate`로 변경(삭제 개념 없음).

## 안전
- 한꺼번에 많이 바꾸는 write — **목록 확인 후** 진행.
