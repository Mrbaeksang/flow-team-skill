# 데일리 브리핑

## 무엇
아침에 읽는 짧은 현황: 챙길 업무·알림·일정. 읽기 전용.

## 기대효과
- 60개 프로젝트에 흩어진 내 상태를 **30초에 한 화면**으로 (직접 클릭하며 뒤질 필요 X)
- 읽기만 해서 **안전** — 워크스페이스를 안 바꿈
- 매일 같은 형식이라 **습관화** 쉬움

## 흐름 (API 조합)
1. `flow.me()` / `flow.myProjects()` — 신원 + 내 프로젝트.
2. 각 프로젝트 `flow.tasks(pid)` → `columns[]` 파싱(TASK_NM/STATUS/WORKER_ID/END_DT). 내 담당만, `END_DT`로 밀림/오늘/임박 분류.
3. `flow.alarms()` → `readYn==="N"`, `mentionYn==="Y"` 우선.
4. `flow.events(오늘 00~23:59)` → 오늘 일정.
5. 합쳐 요약 + 다음 행동 제안.

빠른 실행: `node scripts/brief.mjs [YYYYMMDD]`

## 응답 예시
```
📋 백상현님 데일리 브리핑 — 2026-06-16
담당 업무 70건 (미완료 42) · 안읽은 알림 6 · 오늘 일정 0
🔴 밀린 업무 (1)  · 06-15 레몬베이스 목표 넣어보기 〈Flow 목표관리〉
🟡 오늘 마감 (0)  · 없음
🟢 임박 마감 ~6/19 (1)  · 06-18 [AI] 실습형 교육 3차 〈교육〉
🔔 안읽은 알림 (6)  멘션 0 · 내 담당 6
ℹ️ 마감일 없는 미완료 37건 (마감 기준 판단에서 빠짐)
```

## 배운 것 (실호출 검증)
- **호출량이 큼:** 프로젝트당 `tasks` 1회 → 60+개면 1회 브리핑에 ~60콜. **분당 120 한도(429)** 라 연달아 돌리면 막힘 → `flow.mjs`가 429 자동 재시도.
- **재시도 없으면 과소집계:** rate limit으로 일부 프로젝트 스캔이 조용히 누락되면 미완료가 **23으로 적게** 잡혔음(실제 42). 전체 스캔돼야 정확.
- **마감일이 희소:** 담당 업무의 일부만 `END_DT` 있음 → 마감 기준 섹션은 **"마감없음 N건"을 꼭 같이** 표기(안 그러면 "할 일 없음"으로 오해).
- **상태 판정:** STATUS의 `optionName`은 보드마다 다름. `optionCategory`(0=대기·1=진행·2=완료·3=보류)로 판정이 안전.

## 안전
- 원문 JSON은 빼고 요약. 쓰기 없음.
